📊 Data Infrastructure & Analytics Maturity

Data infrastructure quality determines the speed and accuracy of every AI, pricing, and personalisation initiative. Most GCC carriers have siloed data with no real-time access layer.

Level 4 — Real-Time Data Platform (Tier 1)

Tawuniya (AWS), Bupa Arabia (Azure), Daman (AWS)

  • Unified data lake on AWS S3 / Azure Data Lake with real-time event streaming (Kafka or Kinesis)
  • Data mesh architecture: policy, claims, member, agent data all accessible via governed APIs in <100ms
  • ML feature store: model features pre-computed and cached for real-time scoring at quote/claims FNOL
  • BI stack: Tableau/Power BI with real-time dashboards for ops, actuarial, and digital performance
  • Customer 360: unified member view across policy, claims, wellness, and interaction data

Level 3 — Cloud Data Warehouse (Tier 1/2 boundary)

AXA Gulf, ADNIC, Orient, QIC

  • Cloud data warehouse (Snowflake or Azure Synapse); data refreshed daily or near-real-time
  • Structured reporting via BI tools (Power BI); actuarial models run in batch not real-time
  • Limited ML integration: rules-based fraud detection; no real-time pricing model
  • Customer data partially unified: policy and claims linked but no wellness or interaction layer
  • API access to data limited to internal use; no real-time external data sharing

Level 2 — Siloed Data Marts (Tier 2)

GIG Kuwait, Solidarity BH, Oman Arab

  • Separate data marts per system: policy admin, claims, finance all in different databases with no join layer
  • Reporting via Excel extracts or basic SQL queries; no self-serve BI
  • No ML models in production; analytics limited to descriptive reporting
  • Customer data fragmented: no single policyholder view; renewal data manual export from core system
  • Data quality issues: duplicate records, inconsistent Arabic name encoding, missing fields

Level 1 — Legacy Transactional Only (Tier 3)

Most Kuwait, Oman, smaller BH carriers

  • Core system is the only data store: policy + claims in single transactional database; no analytics layer
  • Reporting = manual Excel extraction by IT team; no self-serve; days to get a management report
  • No data science capability; no ML; no predictive analytics
  • Customer data: paper forms scanned and filed; digital data patchy and uncleaned
  • Cannot do personalisation, propensity modelling, or real-time pricing without a full data platform build
